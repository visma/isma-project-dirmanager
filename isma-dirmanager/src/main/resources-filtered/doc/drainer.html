Drainage de fichiers

Retour à l'index

Copie de la source vers la destination les fichiers manquants.
Cette opération crée les arborescences absentes dans un repertoire nommé _DRAINED_FILES.
Les fichiers déjà existants dans l'arborescence destination ne sont pas dupliqués même si leur localisation est différente.

Signalétique des couleurs

  • Bleu : le fichier n'existe pas de l'autre côté
  • Vert : le fichier a été copié de ce côté
  • Noir : le fichier a été copié de l'autre côté ou il était déjà synchrone
